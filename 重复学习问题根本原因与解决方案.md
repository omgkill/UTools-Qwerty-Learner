# 重复学习问题根本原因分析与解决方案

## 📋 执行摘要

**问题**：重复学习进入后，当前进度没有保存，重新打开界面依旧从头开始。

**根本原因**：架构设计缺陷 - 存在三个独立且不同步的状态源（Redux store、useWordList state、IndexedDB），导致竞态条件和数据不一致。

**解决方案**：引入 `RepeatLearningManager` 统一管理所有状态，采用"同步更新内存 + 异步持久化 + 版本控制"的架构。

---

## 🔍 问题深度分析

### 1. 当前架构的致命缺陷

#### 1.1 三足鼎立的状态管理

```
┌────────────────────────────────────────────────────────┐
│                   重复学习状态                         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  ① store.uiState.isRepeatLearning                     │
│     位置：Redux/Immer store                           │
│     作用：控制 UI 渲染和业务流程                        │
│     问题：更新时机不可靠，与实际状态不同步            │
│                                                        │
│  ② learningWords (useWordList 内部 state)              │
│     位置：useWordList.ts useState                     │
│     作用：实际学习的单词列表                          │
│     问题：与 store 状态没有强制同步机制                │
│                                                        │
│  ③ IndexedDB typingStates 表                          │
│     位置：浏览器数据库                                 │
│     作用：页面刷新后恢复状态                          │
│     问题：异步读取，与 React 渲染周期不同步            │
│                                                        │
└────────────────────────────────────────────────────────┘
```

#### 1.2 竞态条件详解

**场景：用户学到第 10 个词，刷新页面**

```
时间线分析：

T0: currentIndex = 10
    用户正在学习第 10 个单词

T1: useEffect(index.tsx:177) 触发
    依赖项 [learningWords, state.wordListData.index] 变化
    开始异步保存：saveRepeatLearningState(..., currentIndex: 10)
    
T2: 用户刷新页面
    浏览器开始重新加载页面
    
T3: IndexedDB 保存操作完成
    数据写入：{ currentIndex: 10, ... }
    但是！此时页面已经在重新加载
    
T4: 页面重新加载完成
    组件重新挂载
    所有状态重置为初始值
    
T5: 恢复 useEffect(index.tsx:159) 执行
    异步读取 IndexedDB: loadRepeatLearningState()
    
T6: 恢复操作完成
    读取到 currentIndex: 10
    dispatch(SET_CURRENT_INDEX, 10)
    setLearningWords(savedWords)
    
T7: ⚠️ 问题出现！
    React 渲染周期中，其他 useEffect 可能触发
    导致 currentIndex 被重置为 0
    
T8: 新的保存逻辑触发
    useEffect(index.tsx:177) 再次触发
    保存 currentIndex: 0
    
T9: ❌ 进度丢失！
    用户看到从头开始
```

#### 1.3 其他问题场景

**场景 A：跨天失败**
```
23:50 → 用户开始重复学习
00:10 → 第二天，日期检查失败
结果：状态无法恢复 ❌
```

**场景 B：并发覆盖**
```
T0: 学到第 5 个词，触发保存（读取旧数据）
T1: 学到第 6 个词，触发保存（读取旧数据）
T2: 两次保存都基于相同的基础数据
T3: 第二次保存覆盖第一次
结果：currentIndex 停留在 5 ❌
```

**场景 C：切换词库污染**
```
T0: 词库 A 重复学习中（currentIndex: 10）
T1: 切换到词库 B
T2: 词库 B 的恢复逻辑读取到词库 A 的数据
结果：数据混乱 ❌
```

## 💡 根本性解决方案

### 2.1 核心设计原则

```
┌──────────────────────────────────────────────────────┐
│            RepeatLearningManager 设计原则            │
├──────────────────────────────────────────────────────┤
│                                                      │
│  1️⃣ 单一状态源                                       │
│     所有状态都通过 Manager 管理                      │
│     内存状态是唯一真实来源（Single Source of Truth）│
│                                                      │
│  2️⃣ 原子操作                                         │
│     每次更新都是原子的                               │
│     避免并发更新导致的数据不一致                     │
│                                                      │
│  3️⃣ 版本控制                                         │
│     使用 version 字段检测并发更新                    │
│     避免旧数据覆盖新数据                             │
│                                                      │
│  4️⃣ 同步优先                                         │
│     关键操作（读取、更新索引）使用同步逻辑           │
│     异步操作（持久化）不阻塞主流程                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 2.2 新架构设计

```typescript

│                 RepeatLearningManager                   │
│                                                         │
│  ┌───────────────────────────────────────────────────┐ │

│  └───────────────────────────────────────────────────┘ │

│  ┌───────────────────────────────────────────────────┐ │
│  │         原子操作 (Atomic Operations)              │ │
│  │                                                   │ │
│  │  initialize(dictId)                               │ │
│  │    → 同步检查内存                                 │ │
│  │    → 异步加载 IndexedDB                           │ │
│  │    → 更新内存状态                                 │ │
│  │                                                   │ │

│  ┌───────────────────────────────────────────────────┐ │
│  │       IndexedDB (持久化，带版本控制)              │ │
│  │                                                   │ │
│  │  {                                               │ │
│  │    dictId: string                                │ │
│  │    date: string                                  │ │
│  │    isRepeatLearning: boolean                     │ │
│  │    learningWords: WordWithIndex[]                │ │
│  │    currentIndex: number                          │ │
│  │    version: number  ← 防止并发覆盖               │ │
│  │  }                                               │ │
│  └───────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 2.3 数据流对比

**旧架构（有问题）**：
```
用户操作 → useEffect 触发 → 异步保存 → 无版本控制 → 可能覆盖
                ↓
           依赖 React 渲染周期
                ↓
           竞态条件高发
```

**新架构（解决方案）**：
```
用户操作 → Manager.updateIndex() → 同步更新内存 (立即生效)
                                      ↓
                                返回最新状态
                                      ↓
                                异步保存 IndexedDB
                                      ↓
                                版本检查 (防止覆盖)
```
