# 学习配置方案

## 一、推荐方案：固定上限型

### 核心规则

```
┌─────────────────────────────────────────────────────────────┐
│  固定上限型核心规则                                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 每日学习上限：默认 20 个词（可在设置中配置）                │
│                                                             │
│  2. 复习优先：有到期词先复习                                 │
│                                                             │
│  3. 新词配额 = max(0, 每日上限 - 当日已复习数 - 当日已学习新词数)│
│     - 当日已复习数只计入前 N 个（N=每日上限）                  │
│     - 额外复习不计入                                         │
│                                                             │
│  4. 额外复习机制：                                           │
│     - 当到期词 > 每日上限时，复习完上限数量后                │
│     - 用户可自愿继续复习剩余到期词                           │
│     - 额外复习不计入总量，不影响新词配额                     │
│                                                             │
│  5. 已掌握词不计入当日复习计划：                             │
│     - 背单词过程中点击"已掌握"即视为已掌握                 │
│     - 该词不计入当日复习计划                                 │
│     - 不消耗学习配额                                         │
│     - 鼓励用户保持对已掌握单词的复习                         │
│                                                             │
│  6. 无硬性目标：用户想学多少就学多少                         │
│                                                             │
│  7. 巩固模式（可选）：                                       │
│     - 当无到期词且无新词时，可进入巩固模式                   │
│     - 也可直接结束当天学习                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 配置参数

```typescript
export const LEARNING_CONFIG = {
  DAILY_LIMIT: 20,              // 每日学习上限（可配置）
}
```

## 二、学习流程

```
┌─────────────────────────────────────────────────────────────┐
│  固定上限型学习流程                                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  打开应用                                                    │
│      │                                                      │
│      ▼                                                      │
│  检查到期词数量                                              │
│      │                                                      │
│      ▼                                                      │
│  计算新词配额                                                │
│      │                                                      │
│      │  新词配额 = max(0, DAILY_LIMIT - 当日已复习数 - 当日已学习新词数) │
│      │  当日已复习数只计入前 DAILY_LIMIT 个                    │
│      │  背单词中点击"已掌握"不计入当日复习计划               │
│      │                                                      │
│      ▼                                                      │
│  有到期词？                                                  │
│      │                                                      │
│      ├─ 是 → 复习到期词（优先）                              │
│      │      │                                               │
│      │      ├─ 复习数 < DAILY_LIMIT                         │
│      │      │      │                                        │
│      │      │      └─ 继续复习，消耗配额                    │
│      │      │      （点击"已掌握"不计入当日复习计划）        │
│      │      │                                               │
│      │      └─ 复习数 = DAILY_LIMIT 且 还有到期词            │
│      │             │                                        │
│      │             └─ 提示：是否继续额外复习？              │
│      │                    （额外复习不计入总量）             │
│      │                                                      │
│      └─ 否 → 有新词且新词配额>0？                            │
│             │                                                │
│             ├─ 是 → 学习新词（最多 = 新词配额）              │
│             │                                                │
│             └─ 否 → 巩固模式（可选） / 结束当天学习           │
│                                                             │
│  用户可随时退出，进度自动保存                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 三、掌握单词详细机制

### 规则说明

```
┌─────────────────────────────────────────────────────────────┐
│  掌握单词详细机制                                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  当用户点击掌握一个单词时：                                  │
│                                                             │
│  1. 这个单词被标记为掌握（masteryLevel = MASTERED）          │
│                                                             │
│  2. 不占用每日配额（不计入 learnedCount 或 reviewedCount）   │
│                                                             │
│  3. 系统会立即补充一个新词给用户学习                         │
│                                                             │
│  4. 补充的新词才会计入 learnedCount                          │
│                                                             │
│  所以实际流程是：                                            │
│                                                             │
│  第一天：                                                    │
│  - 开始学习 20 个新词                                        │
│  - 学到第 5 个词时点击掌握 → 系统补充 1 个新词（第 21 个）   │
│  - 学到第 15 个词时点击掌握 → 系统补充 1 个新词（第 22 个）  │
│  - 最终：学习了 22 个词，其中 2 个掌握，20 个有效学习        │
│         → learnedCount = 20                                  │
│                                                             │
│  第二天：                                                    │
│  - 开始复习 18 个词（第一天 20 个有效学习的词到期了）        │
│  - 复习到第 3 个词时点击掌握 → 系统补充 1 个新词             │
│  - 复习到第 8 个词时点击掌握 → 系统补充 1 个新词             │
│  - 最终：学习了 20 个词（18 个复习 + 2 个新词）              │
│         其中 2 个掌握，18 个有效复习                         │
│         → reviewedCount = 18                                 │
│                                                             │
│  核心原则：                                                  │
│  - 掌握的单词相当于"删除"，不占用每日配额                   │
│  - 系统始终保持用户学习 N 个"未掌握"的单词                   │
│  - 每日配额 = 新词 + 复习词 = 20 个（不包含掌握词）         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、额外复习机制详解

### 触发条件

```
到期词数量 > DAILY_LIMIT（默认 20）
```

### 机制说明

```
┌─────────────────────────────────────────────────────────────┐
│  额外复习机制                                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  场景：用户有 35 个到期词                                      │
│                                                             │
│  第一步：复习前 20 个（计入总量）                              │
│    - 消耗 20 个学习配额                                        │
│    - 新词配额 = 20 - 20 = 0                                 │
│                                                             │
│  第二步：提示额外复习                                        │
│    - "您还有 15 个单词需要复习，是否继续？"                    │
│    - 用户可选择：继续复习 / 稍后再说                         │
│                                                             │
│  第三步：额外复习（不计入总量）                              │
│    - 复习剩余 15 个词                                          │
│    - 不消耗学习配额                                          │
│    - 不影响新词配额（仍为 0）                                 │
│    - 但可以快速消化堆积                                      │
│                                                             │
│  效果：                                                      │
│    - 用户可以利用空闲时间快速消化堆积                        │
│    - 不会因为堆积而无法学习新词                              │
│    - 复习越多，到期词越少，新词配额恢复越快                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 五、详细模拟

到期词：0 个
新词配额：20 - 0 = 20 个

学习过程：学习单词 1-20（20 个新词）

单词状态：单词 1-20 等级 1，下次复习：1 天后

## 六、测试用例要求

### 掌握单词测试场景

测试用例应模拟真实的业务流程，验证掌握单词机制的正确性：

**测试场景 1：第一天学习 20 个新词，掌握 2 个**

```
初始状态：
- 词库：100 个新词
- 每日上限：20 个

学习过程：
1. 加载 20 个新词（word0-word19）
2. 学到第 5 个词（word4）时点击掌握
   - 调用 handleMasteredFlow
   - 标记 word4 为 MASTERED
   - 调用 incrementMastered
   - 系统补充 word20
   - 学习 word20（计入 learnedCount）
3. 学到第 15 个词（word14）时点击掌握
   - 调用 handleMasteredFlow
   - 标记 word14 为 MASTERED
   - 调用 incrementMastered
   - 系统补充 word21
   - 学习 word21（计入 learnedCount）

预期结果：
- learnedCount = 20（18 个初始词 + 2 个补充词）
- masteredCount = 2（word4, word14）
- 总共学习了 22 个词
```

**测试场景 2：第二天复习 18 个词，掌握 2 个**

```
初始状态：
- 到期词：18 个（第一天 20 个有效学习的词）
- 每日上限：20 个

学习过程：
1. 加载 18 个到期词
2. 复习到第 3 个词时点击掌握
   - 调用 handleMasteredFlow
   - 标记为 MASTERED
   - 调用 incrementMastered
   - 系统补充 word22
   - 学习 word22（计入 learnedCount）
3. 复习到第 8 个词时点击掌握
   - 调用 handleMasteredFlow
   - 标记为 MASTERED
   - 调用 incrementMastered
   - 系统补充 word23
   - 学习 word23（计入 learnedCount）

预期结果：
- reviewedCount = 18（16 个复习词 + 2 个补充词）
- learnedCount = 2（2 个补充的新词）
- masteredCount = 2
- 总共学习了 20 个词
```

**测试场景 3：第二天学习完后重新打开界面**

```
初始状态：
- 第二天学习了 20 个词（18 个复习 + 2 个新词）
- 达到每日上限

预期结果：
- 重新打开界面后，学习类型应为 new（还有新词可学）
- 因为掌握的 2 个词不占用配额，系统还有学习空间
- 可以继续学习 4 个新词（20 - 18 + 2 = 4）
```

### 测试要点

1. **使用 handleMasteredFlow**：测试中应调用 `handleMasteredFlow` 模拟真实的掌握流程
2. **调用 incrementMastered**：掌握单词后必须调用 `incrementMastered` 增加掌握计数
3. **补充新词**：掌握单词后，系统应补充新词，补充的新词需要用户学习完成后才计入 learnedCount
4. **掌握不计入配额**：掌握的单词不计入 learnedCount 或 reviewedCount
5. **验证单词状态**：掌握的单词 masteryLevel 应为 MASTERED（7）
6. **验证学习详情**：统计时应排除掌握的单词

### 测试代码示例

```typescript
it('第一天和第二天：学习 20 词过程中，有两个词点击掌握 -> 验证学习详情', async () => {
  // 第一天：学习 20 个新词，掌握 2 个
  const firstDaySession = await loadTypingSession({...})
  
  for (let i = 0; i < firstDaySession.learningWords.length; i++) {
    const word = firstDaySession.learningWords[i]
    const isMastered = i === 4 || i === 14
    
    if (isMastered) {
      // 模拟点击掌握的逻辑
      const result = await handleMasteredFlow({
        currentWord: { ...word, index: i },
        markAsMastered: (wordName) => wordProgressService.markAsMastered(dictId, wordName),
        getNextNewWord: async () => {
          // 模拟补充新词
          const newWord = wordList[20 + supplementCount]
          supplementCount++
          return { ...newWord, index: 20 + supplementCount }
        },
      })
      
      // 调用 incrementMastered（真实业务中 handleMastered 会调用）
      await dailyRecordService.incrementMastered(dictId)
      
      // 学习补充的新词
      if (result.replacementWord) {
        await wordProgressService.updateProgress(dictId, result.replacementWord.name, true, 0)
        await dailyRecordService.incrementLearned(dictId)
      }
    } else {
      // 正常学习
      await wordProgressService.updateProgress(dictId, word.name, true, 0)
      await dailyRecordService.incrementLearned(dictId)
    }
  }
  
  // 验证结果
  const updatedFirstDayRecord = await dailyRecordService.getTodayRecord(dictId)
  expect(updatedFirstDayRecord.learnedCount).toBe(20)
  expect(updatedFirstDayRecord.masteredCount).toBe(2)
})
```
